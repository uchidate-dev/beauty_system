# Beauty System (ハイエンドサロン向け　独立型予約・業務管理システム)

洗練された顧客体験(UX)と、美容室のリアルな経営課題の解決を両立する、自社導入型の予約・業務管理プラットフォームです。

## ターゲットユーザー

- **店舗規模:** スタッフ 3〜5名規模の独立系ハイエンドサロン
- **解決する課題:** 外部の予約ポータルサイトへの高い掲載料・手数料による利益圧迫と、自社での顧客データ蓄積のハードル。

## 開発背景：「友人の開業」から見えたリアルな葛藤

友人が美容室を開業する中で、「予約管理や集客を外部サービスに依存せざるを得ず、毎月の手数料が経営の重荷になっている」というリアルな悩みを聞いたことが開発のきっかけです。

一方で、自社で独自の予約システムを持つには開発コストや運用のハードルが高く、個人経営のサロンにとっては現実的ではないというジレンマもありました。
「現場でも無理なく使えて、かつ外部依存から少しでも抜け出せる仕組みを、技術で提供できないか？」と考え、実際に友人にヒアリングを行い、現場の業務フローを整理した上で本システムを設計しました。

## 導入による想定成果（Instagram集客との連携）

本システム単体で「新規集客」を完全に代替するのは難しいため、**Instagram等のSNSのプロフィールから直接システムへ繋ぎ、リピーターの囲い込みと新規集客を両立する**運用を想定しています。

- **利益率の大幅な改善:** SNS経由の予約を自社システムで直接受けることで、高額な外部プラットフォームの手数料を削減。
- **失客率の低下:** 画面を切り替えずにリアルタイムで空き状況がわかる仕組みにし、「予約しようとしたが埋まっていた」というストレスによる顧客離脱を防ぎます。
- **機会損失の削減:** 厳密なキャンセル期限(前日23：59まで)を設けることで、ドタキャンを未然に防ぎ、空き枠への新規予約や電話対応を促します。

## こだわり設計(UI/UX × 業務ロジック)

### 1. ユーザー離脱を防ぐ「ストレスフリー」な予約導線

Instagram等のSNSから流入したお客様が迷わず直感的に予約を完了できるよう、ページ遷移を極力減らしたスムーズな操作性を実現。ハイエンドサロンにふさわしい「余白を活かした引き算のデザイン」で入力ストレスを排除しました。

### 2. 店舗の機会損失を防ぐ「厳密なキャンセル制御」

画面上のボタンを隠すだけの簡単なフロント制御ではなく、サーバーサイドでキャンセル期限を厳密にチェックする堅牢な設計を実装し、店舗の不利益を確実に防ぎます。

### 3. 現場の運用ルールを完全再現した「スマートな裏側処理」

「指名なし予約」に対し、お客様には「指名なし」として見せつつ、管理者側では特定スタッフを割り当て、売上集計上は「フリー客」として処理する等、美容室特有の複雑なビジネスルールをシステムに落とし込みました。

## バックエンド実装における技術的課題と解決策

単なるCRUDアプリにとどまらず、実際の店舗運用に耐えうる複雑な予約ロジックと、インフラ環境に依存しない堅牢な実装にこだわりました。

- **動的なタイムライン生成とコマ数計算:**
  ユーザーが選択した複数メニューの「合計所要時間」を算出し、30分単位で必要なコマ数（枠）を自動計算。空き状況の判定およびDBへの予約保存を、時間枠ごとに正確に処理するロジックを構築。
- **排他制御とインフラ最適化:**
  予約確定時のダブルブッキングを防ぐため、満席チェックから保存処理までのトランザクション処理を実装。また、初期デプロイ時のDB（PostgreSQL）の厳格なロック仕様によるデッドロック課題に対し、自らMySQL(JawsDB)環境への移行と設定変更を実施し、確実な予約処理を担保しました。
- **運用を想定した動的なテストデータ生成（Seeder）:**
  いつログインしても「リアルな繁盛店」の運用を体験できるよう、Seeder内で現在時刻からの相対日付を計算。「スタッフの定休日」を判定して予約を弾くロジックを組み込み、矛盾のないデモデータを自動生成しています。

## 使用技術

- **Laravel 11:** 複雑な予約ルールやビジネスロジックを、安全かつ正確に処理するため。
- **Tailwind CSS / Alpine.js:** 洗練されたデザインと、非同期なUI操作（モーダル開閉やタブ切替）を実現するため。
- **Chart.js:** 直感的な管理者ダッシュボードで「月別売上」や「スタッフ別売上」を可視化するため。

## デモ体験について（採用ご担当者様へ）

採用ご担当者様がパスワード入力等の手間なく、スムーズにUI/UXおよび機能をご体験いただけるよう、**ワンクリックでログイン可能なデモ機能**を実装しております。

**【デモ体験の手順】**

1. **アプリURL:** [https://beauty-salon-system-5e46f31a3e68.herokuapp.com](https://beauty-salon-system-5e46f31a3e68.herokuapp.com) にアクセス
2. トップページより「Book Now / オンライン予約」をクリック
3. ゲート画面にて「Login / 会員の方」を選択
4. ログイン画面下部の「デモ体験」エリアより、いずれかのボタンをクリックしてください。
    - **一般のお客様（デモ）:** ユーザー視点での予約フローをご体験いただけます。
    - **サロン管理者（デモ）:** 管理者視点での予約タイムライン確認、スタッフ割り当て、ダッシュボード機能等をご確認いただけます。

> **【運用に関する補足】**
> 自由にデータの追加・更新・削除をお試しください。
> 実際のWebサービス運用を想定し、デモユーザーのデータは**毎日深夜0時に自動リセット（Seederの再実行）**される設計としております。
